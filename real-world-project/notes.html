Disease Ontology Explorer

Objective

Build a Disease Ontology Explorer that allows biologists to search for diseases, explore their hierarchical context, and access related research papers. The application should support the following features:

  • Search for diseases using the EBI Ontology Lookup Service (OLS4) API.
  • Display detailed information about selected diseases, including EFO ID and description.
  • Show the hierarchical context of diseases.
  • Fetch and display related research papers.
  • Provide a user-friendly and responsive interface.

My Approach with the Assistance of Claude AI

  1. I started by setting up a Next.js project with TypeScript for a robust development environment. I chose Next.js for its server-side rendering capabilities and API routes feature, which would be crucial for handling API requests securely.
    • Used create-next-app to initialize the project
    • Set up TypeScript for type safety
    • Implemented the App Router for better routing capabilities
  2. For the UI design, I opted for a clean and modern look using Tailwind CSS and shadcn/ui components. This combination allowed for rapid development of a responsive and visually appealing interface.
    • Installed and configured Tailwind CSS
    • Integrated shadcn/ui for pre-built, customizable components
    • Created a consistent color scheme and typography system
  3. The core functionality was built around efficient data fetching and state management:
    • Implemented API calls to the EBI OLS4 service using the native fetch API
    • Used React hooks (useState, useEffect) for local state management
    • Created custom hooks for reusable logic (e.g., useDebounce for search input)
  4. For the search and display functionality, I focused on creating a smooth user experience:
    • Developed an autocomplete search bar with real-time results
    • Implemented error handling and loading states for better UX
    • Created a detailed view for selected diseases with expandable sections
  5. The hierarchical display of diseases was a key feature:
    • Fetched hierarchical data from the OLS4 API
    • Implemented an expandable tree-like structure for easy navigation
    • Added animations for a more engaging user experience
  6. For the research papers feature, I tried to integrate Google Scholar papers via web scraping using Cheerio. However, I was unable to fetch the data, so I decided to leave it out.
  7. Throughout the development process, I ensured the application was responsive and accessible:
    • Used Tailwind's responsive classes for different screen sizes
    • Implemented keyboard navigation and proper ARIA attributes
    • Tested the application across various devices and browsers
