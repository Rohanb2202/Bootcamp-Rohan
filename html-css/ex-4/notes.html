Notes

Exercise-4

Level 1:

Create a centered webpage layout using %, vh, and vw. Center a content block horizontally and vertically.

Level 2:

Use min(), max(), and clamp() for font sizes and widths. Style the page content to scale proportionally across different screen sizes.

Level 3:

Implement continuous responsiveness by avoiding fixed breakpoints. Use advanced responsive techniques like flexible grids and fluid typography.

Deliverables:

Responsive solution.html (one or more, as needed) with updated CSS. Notes.html explaining the choice of units and responsive strategies used.

My Approach

I approached the responsive design task methodically, addressing each level's requirements with attention to scalability and user experience. Here's how I structured my solution:

Level 1

I utilized percentages (%), viewport height (vh), and viewport width (vw) units to create a fluid layout. The content block was centered horizontally and vertically using a combination of display: flex; and justify-content: center; align-items: center;.

Level 2

To achieve proportional scaling, I implemented min(), max(), and clamp() for defining font sizes and element widths. This allowed content to adapt seamlessly to various screen sizes while maintaining readability and layout integrity.

Level 3

For advanced responsiveness, I avoided fixed breakpoints by leveraging fluid grids and Key frames for fade-in animation.

Challenges and Strategies

Adapting to different screen sizes required testing and iterative refinement. I ensured that the design remained intuitive by prioritizing content hierarchy and balance. Modern CSS techniques like grid-template-columns and font-size with clamp() were particularly effective.
