Exercise 1

Objective

You have a list of research papers, which you can find on platforms like Kaggle. Your task is to build a UI to filter this data. The application should support the following features:

  • Retrieve 1,000 papers from the backend.
  • Display them in a list of cards.
  • Implement a filter widget to filter the data based on specific criteria.
  • Add a search widget to search across all columns.
  • Provide functionality to export the filtered data as a zip file.

My Approach

  1. First and foremost, I focused on the core functionality. I decided to use vanilla JavaScript and modular code organization to keep things lightweight and maintainable.
    I split the code into separate files:
    • A clean HTML structure in index.html
    • styles.css for the visually pleasing UI
    • Core functionality in script.js
  2. For the design phase, I went with a modern, card-based layout that's both functional and visually appealing. I used CSS variables for consistent theming and made extensive use of Flexbox and Grid for responsive layouts. The color scheme focuses on readability with a professional blue palette. For this part I got assitance from our beloved friend Claude AI who really did some wonder on the stylesheet.
  3. Since the data management is a crucial aspect, I built a system that:
    • fetches the research papers data from JSON file.
    • Maintained a separate arrays for all the papers and filtered results.
    • Updates the display efficiently when the filters are been applied.
    • Handles pagination calculations thoroughly.
  4. The filtering system was a key priority. I implemented:
    • A real-time search functions that filters across multiple fields.
    • Additionally, separate filters for institutions, journals and years.
    • A clean way to reset all the applied filters so that the page reverts back to its initial state
    • Finally export functionality for the filtered results
