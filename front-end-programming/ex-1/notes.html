Exercise 1: Basic Front End

Question

Objective: Develop a front-end project using pure HTML, CSS, and JavaScript. Progressively introduce modularity, file separation, and libraries for handling data.

Problem to Solve: Build a dynamic student management dashboard that displays student records in a table. This table will be enhanced over the exercises by adding new features, libraries, and tools. Start small and incrementally improve.

Steps:

  1. Single-File Application (HTML, CSS, JS):
    • Create a single HTML file with inline CSS and JavaScript.
    • Add a button labeled "Add Student" that performs the following:
      • DOM Manipulation: Adds a new row to an existing table element.
      • CSS Manipulation: Dynamically adds a CSS class highlight to the newly added row.
      • Google Font Integration: Changes the table font dynamically to a Google Font.
    • Display browser information below the table.
  2. Separation of Concerns (JS in External File):
    • Move JavaScript to an external file.
    • Load student records from a JSON file and render the table dynamically.
  3. Using a Library for Data Presentation:
    • Use AG Grid for rendering student records with enhancements like highlighting rows, sorting, and filtering.

My Approach(Haven't Used any LLM)

  1. Single-File Application:
    • Created a single HTML file with inline CSS and JavaScript to handle all functionality.
    • Added an "Add Student" button with event listeners to dynamically manipulate the DOM and add rows to the table.
    • Defined a highlight class in CSS to style newly added rows and applied it dynamically via JavaScript.
    • Implemented a button to change the font dynamically by loading Google Fonts on click.
    • Used the navigator and window objects to display browser details below the table.
  2. Separation of Concerns:
    • Moved JavaScript to an external file script.js for better maintainability.
    • Created a JSON file students.json to store initial student data and dynamically loaded it into the table using fetch.
    • Separated code into modules:
      • data-loader.js: Responsible for fetching and parsing JSON data.
      • table-renderer.js: Responsible for generating and rendering table rows.
  3. Using a Library:
    • Integrated AG Grid for enhanced data presentation.
    • Loaded student data from a CSV file using PapaParse and rendered it in the AG Grid table.
    • Added features like row highlighting for grade "A", column sorting, and filtering using AG Grid's built-in options.
