Exercise 2: Using Tools

Question

Objective: Learn how to package and structure projects using TypeScript. Follow a development life cycle: Development → Testing → Production.

Steps:

  1. Set Up a TypeScript Project:
    • Initialize a new project using a package manager like npm.
    • Configure TypeScript using a tsconfig.json file.
  2. Three Problems to Solve:
    • Problem 1: Load a JSON file, render a table dynamically, and include sorting functionality.
    • Problem 2: Fetch data from a CSV file using PapaParse and enhance the table with column filtering and row highlighting.
    • Problem 3: Integrate a build tool (e.g., tsc for transpilation), package the project, and ensure it works in both development and production environments.
  3. Dev, Test, and Production Life Cycle:
    • Use Vite in your package.json for:
      • Development server with live reload.
      • Linting and type-checking during development.
      • Production build with bundling.

My Approach

  1. Set Up a TypeScript Project:
    • Initialized a new project using npm init and installed TypeScript using npm install typescript.
    • Created a tsconfig.json file to configure TypeScript settings, enabling strict mode and specifying output directories.
  2. Problem 1:
    • Loaded a JSON file using TypeScript's built-in fs module.
    • Dynamically generated an HTML table and implemented sorting functionality using TypeScript's type-safe methods.
  3. Problem 2:
    • Used PapaParse to fetch and parse data from a CSV file.
    • Enhanced the table with:
      • Column filtering using dropdown menus.
      • Row highlighting for specific conditions, such as grades above a threshold.
  4. Problem 3:
    • Set up tsc for TypeScript transpilation to JavaScript.
    • Packaged the project using npm scripts, ensuring compatibility across environments.
  5. Dev, Test, and Production Life Cycle:
    • Configured Vite for a development server with live reload and integrated ESLint for linting and type-checking during development.
    • Set up a production build process using Vite's bundling capabilities.
    • Tested the application across all environments to ensure smooth transitions from development to production.
