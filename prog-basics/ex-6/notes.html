Exercise 6: YAML Configuration (config)

Level 1

  1. Problem Description:
    • Install the config library and create a default.yaml configuration file.
    • Load and use configuration values in the program.
  2. Exit Criteria:
    • The program reads values from the YAML file.

Level 2

  1. Problem Description:
    • Add multiple environments (e.g., development, production) to the configuration.
    • Load the appropriate configuration based on the environment.
  2. Exit Criteria:
    • The program switches configurations based on the environment.

Level 3

  1. Problem Description:
    • Use the configuration to dynamically adjust program behavior (e.g., logging levels).
    • Use the configuration file to specify additional commands and also functions and files where the commands are implemented.
    • Dynamically load the functions in your code, extending the functionality. For example, load the functions called sin and cos from a new file specified in the configuration file.
  2. Exit Criteria:
    • Configuration drives program behavior dynamically.
    • New functionality can be added to the code without modifying the old code.

Approach

The following steps were taken to implement YAML-based configuration management:

  1. Level 1:
    • Installed the config library using npm install config js-yaml.
    • Created a default.yaml file with basic configuration values such as server port, database details, etc.
    • Loaded the configuration into the program using require('config') and used the values to set up the program's behavior.
  2. Level 2:
    • Created additional environment-specific configuration files such as development.yaml and production.yaml.
    • Set the NODE_ENV environment variable to switch between configurations.
    • Tested the program to ensure it loaded the correct configuration based on the environment.
  3. Level 3:
    • Enhanced the configuration to include logging levels and dynamically adjustable program behavior.
    • Specified additional commands, functions like math-function.js and custom-funtion.js, and their respective file paths in the configuration file.
    • Used require() dynamically to load the specified functions and integrate them into the program.
    • Tested the program to ensure that new functionality could be added via the configuration file without modifying the existing codebase.

This approach ensures modularity, scalability, and maintainability of the codebase by leveraging YAML-based configuration files.
