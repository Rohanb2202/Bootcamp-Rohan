Exercise 4: Debugging (debug)

Level 1

  1. Problem Description:
    • Install the debug package and set up a namespace for debugging.
    • Use the debug function to log messages in a single module.
  2. Exit Criteria:
    • Debug messages are displayed when the DEBUG environment variable is set.

Level 2

  1. Problem Description:
    • Extend the program with multiple modules, each having its own debug namespace.
    • Use environment variables to selectively enable debugging for specific modules.
  2. Exit Criteria:
    • Debugging can be turned on/off for individual modules.

Level 3

  1. Problem Description:
    • Combine debug with winston for structured debugging and logging.
    • Use both tools to debug and log a complex program scenario.
    • Enhance the compute program by making it interactive.
  2. Exit Criteria:
    • Debug and log messages work seamlessly together.

Approach

The following steps were taken to implement selective debugging:

  1. Level 1:
    • Installed the debug package using npm install debug.
    • Created a single module with a unique debug namespace (e.g., my-module).
    • Used the debug function to log messages at key points in the module.
    • Tested the setup by setting the DEBUG environment variable to the namespace and running the script.
  2. Level 2:
    • Extended the program to include multiple modules, each with its own debug namespace (e.g., moduleA, moduleB).
    • Configured the environment variable to selectively enable debugging for specific modules (e.g., DEBUG=myapp:module2).
    • Verified that debug messages were displayed only for the enabled namespaces.
  3. Level 3:
    • Integrated winston for structured logging alongside debug.
    • Enhanced the compute program by making it interactive, allowing users to input operations and numbers via the command line.
    • Used debug for selective debugging and winston for permanent logging of important events.
    • Tested the program to ensure debug and log messages worked seamlessly together, providing both real-time and structured logs.

This approach ensures efficient debugging and structured logging for scalable and maintainable applications.
