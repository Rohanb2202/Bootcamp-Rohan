Exercise 2: Command-Line Program

Level 1

  1. Problem Description:
    • Create a script that calculates the sum of numbers passed as command-line arguments.
    • Use process.argv to parse and handle the arguments.
    • Include error handling for invalid inputs (e.g., non-numeric values).
  2. Exit Criteria:
    • Running the script with valid inputs returns the correct sum.
    • Invalid inputs result in meaningful error messages.

Level 2

  1. Problem Description:
    • Enhance the script to accept a --multiply flag to multiply the numbers instead of adding them.
    • Update the help text to include usage instructions.
  2. Exit Criteria:
    • The --multiply flag works as expected.
    • Usage instructions are clear and correct.

Level 3

  1. Problem Description:
    • Modularize the program by creating a separate file (mathutils.js) for the add and multiply functions.
    • Take the operation followed by operands as the arguments (e.g., command add 1 2 or command multiply 3 4).
    • Create a script using the bin part of package.json. Use npm link to test the script.
  2. Exit Criteria:
    • The program uses functions from mathutils.js for calculations.
    • The module is reusable in other scripts.
    • The script can be executed as a global command.

Approach

To implement this exercise, the following steps were taken:

  1. Level 1:
    • Created a Node.js script to handle command-line arguments using process.argv.
    • Parsed the arguments and calculated their sum, ensuring to handle invalid inputs with appropriate error messages.
    • Tested the script with both valid and invalid inputs to ensure robustness.
  2. Level 2:
    • Enhanced the script to recognize a --multiply flag for multiplication operations.
    • Updated the help text to clearly document the usage instructions for addition and multiplication operations.
    • Validated the functionality of the flag with various input scenarios.
  3. Level 3:
    • Extracted the add and multiply functions into a separate module (mathutils.js).
    • Refactored the main script to use the exported functions from mathutils.js.
    • Configured the script as a global command by adding a bin field in package.json and tested it using npm link.
    • Verified that the script could be reused and executed globally with proper input handling.

This approach ensures the program is modular, reusable, and follows best practices for command-line utility development.
