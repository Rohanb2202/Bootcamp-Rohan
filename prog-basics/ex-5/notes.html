Exercise 5: Managing Environment Variables (dotenv)

Level 1

  1. Problem Description:
    • Install the dotenv library and create a .env file with configuration variables (e.g., PORT, DB_HOST).
    • Load the variables into the program and use them for configuration.
  2. Exit Criteria:
    • The program reads and uses values from .env.

Level 2

  1. Problem Description:
    • Add fallback values for missing environment variables.
    • Log a warning when a variable is missing.
  2. Exit Criteria:
    • The program handles missing variables gracefully.

Level 3

  1. Problem Description:
    • Use environment variables to configure a database connection and API credentials. (You can use a SQLite file-based database for this exercise.)
    • Securely log sensitive configuration details.
  2. Exit Criteria:
    • Sensitive information is handled securely.

Approach

The following steps were taken to implement environment variable management:

  1. Level 1:
    • Installed the dotenv library using npm install dotenv.
    • Created a .env file with configuration variables such as PORT and DB_HOST.
    • Loaded the environment variables in the program using require('dotenv').config().
    • Used the variables in the program to configure server ports and database hosts.
  2. Level 2:
    • Added fallback values for missing environment variables using process.env.VAR_NAME || 'default_value'.
    • Logged warnings using winston when essential variables were missing.
    • Validated the program's ability to function with partial or missing configurations.
  3. Level 3:
    • Configured a SQLite database connection using environment variables for the file path and credentials.
    • Added API credentials (e.g., supersecretapikey) in the .env file and used them securely within the program.
    • Used structured logging to securely log configuration details without exposing sensitive data (e.g., masking parts of the API key).
    • Tested the program for security by ensuring sensitive information was not logged or exposed inappropriately.

This approach ensures the secure and efficient management of configuration through environment variables, enhancing scalability and security.
