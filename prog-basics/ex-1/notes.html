Exercise 1: Setting Up npm

Level 1

  1. Problem Description:
    • Initialize a new Node.js project using npm init.
    • Add a start script to run a basic index.js file.
    • Add a dev script to run the same file with debugging output (DEBUG=* node index.js).
  2. Exit Criteria:
    • Running npm start executes index.js.
    • Running npm run dev shows debugging output.

Level 2

  1. Problem Description:
    • Install the chalk package and use it in index.js to style console outputs.
    • Add nodemon as a dev dependency and use it to automatically restart the program during development.
  2. Exit Criteria:
    • index.js outputs styled messages using chalk.
    • Changes to index.js automatically restart when using nodemon.

Level 3

  1. Problem Description:
    • Add a prestart script to ensure node_modules is available before running the program.
    • Test by removing node_modules and ensuring npm start runs successfully after reinstalling dependencies.
  2. Exit Criteria:
    • prestart script checks for node_modules and reinstalls dependencies if missing.

Approach

To complete the exercise, the following steps were taken:

  1. Level 1:
    • Initialized a new Node.js project using npm init, providing default answers to streamline the setup process.
    • Created a basic index.js file with a simple console.log message for testing.
    • Added scripts in package.json to enable easy execution: start for running the file and dev for debugging output.
    • Validated the functionality by running npm start and npm run dev.
  2. Level 2:
    • Installed the chalk package and modified index.js to output styled messages using it.
    • Installed nodemon as a development dependency and updated the dev script to use it for automatic restarts.
    • Tested by modifying index.js to confirm that nodemon detected changes and restarted the program automatically.
  3. Level 3:
    • Created a prestart script in package.json to check for the existence of node_modules.
    • Simulated the absence of node_modules by deleting the folder and confirmed that npm start reinstalled dependencies as needed.
    • Ensured seamless execution by verifying that the program ran correctly after the dependencies were reinstalled.

Throughout the process, attention was paid to using best practices for dependency management and script configuration, ensuring a clean and reusable setup.
