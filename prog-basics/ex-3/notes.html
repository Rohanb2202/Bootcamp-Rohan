Exercise 3: Using Logging (Winston)

Level 1

  1. Problem Description:
    • Install winston and create a logger with separate log levels (info, warn, error).
    • Log events such as "Program started" and "Operation successful".
    • Add a few more functions like divide. Division by zero should be logged as a fatal error.
  2. Exit Criteria:
    • Logs are output to the console with appropriate log levels.

Level 2

  1. Problem Description:
    • Add file transport to log messages to a file.
    • Customize the log format to include timestamps.
  2. Exit Criteria:
    • Logs are written to a file with timestamps.
    • The format is clear and readable.

Approach

The following steps were followed to implement structured logging:

  1. Level 1:
    • Installed the winston library using npm install winston.
    • Created a basic logger with three levels: info, warn, and error.
    • Added a divide function, ensuring division by zero triggers a fatal error log.
  2. Level 2:
    • Enhanced the logger by adding file transport to save logs to a file for persistent storage.
    • Customized the log format to include timestamps using winston.format.combine with winston.format.timestamp and winston.format.printf.
    • Validated that logs were being written to the file in a clear and readable format.
    • Ran various tests to ensure that both console and file logging were functioning correctly with timestamps.

By following these steps, the logging implementation became robust, scalable, and suited for real-world applications.
